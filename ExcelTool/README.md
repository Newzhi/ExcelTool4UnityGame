# ExcelTool

Excel 导表工具（MVP）。读取 `TestExcels/` 下的 `.xlsx` 配置表，校验后生成 C# 代码（`*Meta.cs`）和二进制数据（`*.bytes`）。

工具本身不关心 Unity 工程位置，所有输入输出路径由 `export-config.json` 配置。

---

| 依赖 | 版本 |
|---|---|

| IDE（可选） | JetBrains Rider / VS / VS Code |

## Excel 表格式

A 列为**行标记**，B 列起为字段内容。数据从 `##desc` 的下一行开始。

| A 列标记 | 必填 | 说明 |
|---|---|---|
| `##var` | ✅ | 字段名（小写），如 `id`、`name` |
| `##type` | ✅ | 字段类型 |
| `##Alias` | 可选 | 中文别名，写入 XML 注释 |
| `##desc` | 可选 | 字段说明，写入 XML 注释 |

**示例（`Hero.xlsx`）：**

| A | B | C | D |
|---|---|---|---|
| ##var | id | name | hp |
| ##type | uint | string | float |
| ##Alias | ID | 名字 | 生命值 |
| ##desc | 唯一主键 | 名字 | |
| | 10000 | nick | 100 |
| | 10001 | leo | 150 |

**命名规则（方式 A）：**

- 文件名 = 逻辑表名：`Hero.xlsx` → 表名 `Hero`
- 输出代码：`HeroMeta.cs`
- 输出数据：`Hero.bytes`

**支持的类型：**

| ##type | C# 类型 |
|---|---|
| `int` | `int` |
| `uint` | `uint` |
| `long` | `long` |
| `float` | `float` |
| `double` | `double` |
| `bool` | `bool` |
| `string` | `string` |
| `int[]` | `int[]` |
| `uint[]` | `uint[]` |
| `long[]` | `long[]` |
| `float[]` | `float[]` |
| `string[]` | `string[]` |
| `bool[]` | `bool[]` |

数组单元格用 `|` 分隔，空单元格表示空数组。详见 `.cursor/rules/exceltool-types.mdc`。

## 快速开始

### 方式一：双击 .bat（推荐，无需手敲命令）

在资源管理器中进入 `ExcelTool/` 目录：

| 脚本 | 作用 |
|---|---|
| **`export.bat`** | 执行导表，输出到 `TargetFonder/Metas` 与 `TargetFonder/Codes` |
| **`create-sample.bat`** | 生成 `TestExcels/Hero.xlsx` 样例表 |

首次使用顺序：`create-sample.bat` → `export.bat`

> 前提：本机已安装 .NET SDK 8.0，且 `dotnet` 命令在 PATH 中可用。

### 方式二：命令行

#### 1. 打开项目

用 Rider 打开 `ExcelTool/ExcelTool.sln`。

建议将 **Working directory** 设为 `ExcelTool/`（与 `export-config.json` 同级）。

#### 2. 准备测试表

**方式 A：** 使用自带的样例生成命令

```powershell
cd ExcelTool\ExcelTool
dotnet run -- --create-sample
```

会在 `TestExcels/Hero.xlsx` 创建样例表。

**方式 B：** 手动在 `TestExcels/` 下新建符合上述格式的 `.xlsx`。

#### 3. 执行导表

```powershell
cd ExcelTool\ExcelTool
dotnet run
```

成功输出示例：

```
发现 1 张表: Hero
[OK] Hero
全部导出完成。
```

生成文件：

- `TargetFonder/Metas/HeroMeta.cs`
- `TargetFonder/Codes/Hero.bytes`

### 4. 指定配置文件（可选）

```powershell
dotnet run -- "D:\MyProject\export-config.json"
```

## 配置文件说明

`export-config.json` 放在 `ExcelTool/` 目录，路径均**相对于配置文件所在目录**解析。

```json
{
  "excelRoot": "../TestExcels",
  "codeOutput": "../TargetFonder/Metas",
  "dataOutput": "../TargetFonder/Codes",
  "namespace": "Game.Config",
  "tableDiscovery": "scanAll",
  "excludeTables": ["_Template"],
  "includeTables": null
}
```

| 字段 | 说明 |
|---|---|
| `excelRoot` | Excel 源表目录 |
| `codeOutput` | 生成的 `*Meta.cs` 输出目录（`Metas/`） |
| `dataOutput` | 生成的 `*.bytes` 输出目录（`Codes/`） |
| `namespace` | 生成 C# 代码的命名空间 |
| `tableDiscovery` | `scanAll` = 自动扫描目录下所有 xlsx；`explicit` = 使用 `tables` 列表 |
| `excludeTables` | 扫描时排除的表名（如模板表 `_Template`） |
| `includeTables` | 非空时只处理指定表（局部导表）；`null` 表示处理全部 |
| `tables` | 仅 `tableDiscovery` 为 `explicit` 时生效 |

**接入真实 Unity 工程时**，只需修改三个路径，例如：

```json
{
  "excelRoot": "D:/Design/Excel",
  "codeOutput": "D:/MyGame/Assets/Scripts/Generated/Meta",
  "dataOutput": "D:/MyGame/Assets/StreamingAssets/Tables"
}
```

### 新增配置表

往 `excelRoot` 目录放入 `Item.xlsx`，再执行 `dotnet run` 即可，**无需修改配置**。

### 模板表

以 `_` 开头的表可放入 `excludeTables` 避免被导出，例如 `_Template.xlsx`。

## 输出格式

### 生成的 C# 代码

```csharp
// <auto-generated> 此文件由 ExcelTool 自动生成，请勿手动修改。

namespace Game.Config;

public sealed class HeroMeta
{
    public uint Id;
    public string Name;
    public float Hp;
}
```

- 类名：`{表名}Meta`
- 字段名：`##var` 转 PascalCase（`id` → `Id`）
- `##Alias` / `##desc` 生成 `/// <summary>` 注释

### 二进制格式（`.bytes`）

| 偏移 | 内容 | 大小 |
|---|---|---|
| 0 | Magic `"XTBL"` | 4 bytes |
| 4 | Version | uint16 = 1 |
| 6 | RowCount | int32 |
| 10 | ColCount | byte |
| 11 | ColTypes | ColCount × byte |
| — | Rows | 按列顺序序列化 |

**类型 ID：**

| ID | 类型 | 编码 |
|---|---|---|
| 1 | uint | 4 bytes |
| 2 | string | int32 长度 + UTF-8 |
| 3 | float | 4 bytes |
| 4 | int | 4 bytes |
| 5 | long | 8 bytes |
| 6 | double | 8 bytes |
| 7 | bool | 1 byte |
| 11 | int[] | int32 长度 + 各 int |
| 12 | uint[] | int32 长度 + 各 uint |
| 13 | float[] | int32 长度 + 各 float |
| 14 | string[] | int32 长度 + 各 string |
| 15 | long[] | int32 长度 + 各 long |
| 16 | bool[] | int32 长度 + 各 bool |

## 源码架构

```
Program.cs              入口，加载配置，调用 Pipeline
Pipeline/
  ExportPipeline.cs     串联：发现表 → 读 → 校验 → 生成
Config/
  ExportConfig.cs       配置模型
  ExportConfigLoader.cs 加载 JSON，解析绝对路径
  TableDiscovery.cs     自动扫描 / 手动列表
Models/
  FieldType.cs          字段类型枚举
  ColumnDef.cs          列定义
  TableData.cs          表数据（中间结构）
  ExportResult.cs       导出结果
Excel/
  ExcelReader.cs        MiniExcel 读取原始行
  SchemaParser.cs       解析 ##var 等元数据行
Validate/
  TypeConverter.cs      字符串 → 强类型
  TableValidator.cs     校验规则
CodeGen/
  MetaCodeGenerator.cs  生成 *Meta.cs
Export/
  BinaryTypeCodec.cs    单值二进制编码
  BinaryExporter.cs     写出 .bytes
```

**数据流：**

```
export-config.json
  → TableDiscovery（扫描 TestExcels/*.xlsx）
  → ExcelReader（读 xlsx）
  → SchemaParser（解析 ## 行）
  → TableValidator（校验 + 类型转换）
  → MetaCodeGenerator（写 Metas/*.cs）
  → BinaryExporter（写 Codes/*.bytes）
```

**校验规则（MVP）：**

- 必须有 `##var` 和 `##type` 行，且列数一致
- 字段名非空、合法、不重复
- 类型必须在支持列表内
- 数据格可按类型解析
- 第一列 `id` 不可重复
- 任一表失败则退出码为 1，不生成该表输出

## 如何扩展

### 1. 新增字段类型（如 `int`、`bool`、`int[]`）

需要同步修改以下文件：

| 文件 | 改动 |
|---|---|
| `Models/FieldType.cs` | 添加枚举值 |
| `Excel/SchemaParser.cs` | `ParseFieldType()` 识别新类型字符串 |
| `Validate/TypeConverter.cs` | 添加解析逻辑 |
| `CodeGen/MetaCodeGenerator.cs` | `ToCSharpType()` 映射 C# 类型 |
| `Export/BinaryTypeCodec.cs` | 添加写入逻辑和类型 ID |

> 每新增一种类型，工具端编码和 Unity 端解码必须保持一致。

### 2. 新增校验规则

在 `Validate/TableValidator.cs` 的 `ValidateAndConvert()` 中添加，例如：

- 引用表检查（`itemId` 必须存在于 `Item` 表）
- 数值范围检查（`hp > 0`）
- 枚举值合法性

### 3. 生成 Table 加载器

当前 MVP 只生成数据类 `HeroMeta`。若需要 `HeroMetaTable.Get(id)` 等 API：

- 扩展 `CodeGen/MetaCodeGenerator.cs`，或新建 `TableCodeGenerator.cs`
- 在 Unity 侧实现 `BinaryTableLoader` 读取 `.bytes`

### 4. 导出前清理旧文件

从 `TestExcels` 删除表后，`TargetFonder` 中可能残留旧的 `.cs` / `.bytes`。

扩展方式：在 `ExportPipeline.Run()` 开头添加清理逻辑，或在配置中增加 `cleanOutputBeforeExport: true`。

### 5. 支持子目录扫描

当前 `TableDiscovery` 只扫描 `excelRoot` 顶层。若 Excel 按模块分子文件夹：

- 修改 `TableDiscovery.DiscoverByScan()` 的 `SearchOption` 为 `AllDirectories`
- 注意表名与路径的映射规则

### 6. 拆分多项目

代码量增大后，可拆为：

```
ExcelTool.Core/    # Models + Excel + Validate + CodeGen + Export
ExcelTool.CLI/     # Program + Pipeline + Config
```

MVP 阶段单项目即可。

### 7. 发布为独立 exe

```powershell
cd ExcelTool
dotnet publish ExcelTool -c Release -r win-x64 -o ./publish
```

将 `publish/` 目录和 `export-config.json` 发给使用者，对方只需安装 .NET 8 Runtime。

## 常见问题

**Q: 提示找不到 `export-config.json`**

将 Rider Working directory 设为 `ExcelTool/`，或通过参数传入配置文件路径。

**Q: 提示 Excel 源目录不存在**

检查 `export-config.json` 中 `excelRoot` 的相对路径是否正确（相对于配置文件所在目录）。

**Q: 加了新表但没导出**

确认文件名是 `.xlsx`（不是 `.xls`），且不在 `excludeTables` 列表中。

**Q: Excel 临时文件干扰**

工具已自动忽略 `~$` 开头的临时文件。

**Q: 从 TestExcels 删表后旧输出还在**

MVP 不会自动清理，需手动删除 `TargetFonder` 中对应文件，或自行实现清理逻辑。
